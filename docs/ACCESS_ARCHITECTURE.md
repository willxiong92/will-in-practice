# 个人网站访问与权限架构

> 状态：生产已发布，公开、团队与管理员权限线上验收通过
> 更新日期：2026-07-16

## 1. 产品定位

个人网站仍是公开最佳实践发布站，不改造成内部知识库或后台工作台。

权限只用于承接团队协作入口和管理员状态页：

| 身份 | 可访问范围 |
|---|---|
| 公开访客 | 首页、五个领域、实践库、搜索、关于、全部 `approved + public` 正文 |
| 团队成员 | 公开内容 + `/team/` 团队协作入口 |
| 管理员 | 公开内容 + `/team/` + `/admin/` 权限与发布状态页 |

## 2. 内容安全边界

- `private`、`team`、`review_required` 内容继续禁止进入正式构建。
- 登录权限不能替代内容门禁。
- P1-P4、客户摸底表、客户 KP、合同、账号、内部制度与本机路径默认不进入公开站。
- `/team/` 首版只提供安全的协作导航、公开 FDE 内容入口和内容边界说明，不包含内部正文。
- GitHub Pages 不具备 Cloudflare Worker 权限能力，因此回退站只允许出现公开安全内容。

## 3. 路由与权限

| 路由 | 访客 | 团队 | 管理员 |
|---|---:|---:|---:|
| `/`、公开内容路由 | 允许 | 允许 | 允许 |
| `/access/` | 允许 | 允许 | 允许 |
| `/api/auth/*` | 按接口规则 | 按接口规则 | 按接口规则 |
| `/team/` | 跳转登录 | 允许 | 允许 |
| `/admin/` | 跳转登录 | 拒绝 | 允许 |
| `/api/admin/*` | 拒绝 | 拒绝 | 允许 |

## 4. 鉴权实现

- Cloudflare Pages Advanced Mode：`public/_worker.js`。
- 服务端 Secret：`TEAM_PASSWORD`、`ADMIN_USER`、`ADMIN_PASSWORD`、`SESSION_SECRET`。
- 登录成功后写入 HMAC 签名 Cookie。
- Cookie：`HttpOnly`、`Secure`、`SameSite=Lax`、12 小时有效。
- 团队与管理员身份分离。
- 登录接口校验同源、限制请求体大小，并返回非技术化错误。
- 不在源码、HTML、GitHub Actions 或公开构建中写入口令。
- 可选 KV：`WIP_ACCESS_AUDIT`，只记录脱敏访问事件；未绑定时权限仍可运行。

## 5. 视觉方向

参考陪跑 AI 工作台的企业级浅蓝、白色表面、清晰角色入口和正式文案，但保留个人网站现有编辑型知识站风格。

- 公开内容页不整体改造成后台界面。
- 登录页采用左右分区：左侧说明内容边界，右侧提供团队/管理员入口。
- 团队页采用任务入口和公开 FDE 内容精选，不使用密集 Dashboard。
- 管理员页采用紧凑状态清单，不制造并不存在的管理功能。
- 单一钴蓝强调色、统一圆角、低动效、支持键盘与 reduced motion。
- 登录、团队与管理员页面统一 `noindex, nofollow`。

## 6. 首版验收标准

- 未登录可正常访问所有公开文章。
- 未登录访问 `/team/` 会跳转 `/access/?next=/team/`。
- 团队登录可进入 `/team/`，不能进入 `/admin/`。
- 管理员登录可进入 `/team/` 和 `/admin/`。
- 退出后受保护页面重新要求登录。
- 未配置 Secret 时只影响受保护入口，不影响公开站。
- 公开构建仍只包含 78 篇已批准公开稿，内容门禁、dist 门禁和链接门禁通过。
- GitHub Pages 回退构建通过，受控页面不得包含敏感信息。
- 页面在桌面和移动端可用，键盘焦点清晰，尊重 reduced motion。

## 7. 暂不实现

- 在线修改文章或内容审批。
- 把 `team/private` Markdown 打包进生产站。
- 成员独立账号、云端项目库、文件上传与数据库。
- 在管理员页直接修改 Cloudflare Secret。
- 将访问权限等同于公司级身份权限体系。
