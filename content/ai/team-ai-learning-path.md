# 团队 AI 基础学习路径：约 2 小时可验收

**目标：** 能判断任务是否适合用 AI，能把任务与边界说清，能按风险选用方式，能对结果做事实与风险复核。

| 6 练习验收 | 20′ | 三项练习 | 见下 |

## 场景 SOP（速查）

| 场景 | 步骤要点 | 合格 |
|---|---|---|
| 客户话术 | 对象、阶段、语气、禁区；标待确认 | 可复制、无越权承诺 |
| 资料总结 | 先目录/摘要，再追问遗漏 | 结论可回原文 |
| 事实核验 | 官方来源、日期、不确定项 | 不把旧信息当现状 |
| 客户材料分析 | **先脱敏**，再总结与下一步 | 关键判断负责人复核 |
| 复盘纪要 | 事实→问题→原因→动作→责任人 | 有检查日 |
| 经验沉淀 | 目标/步骤/规则/模板 | 新人可照做 → [Skill](/practice/skill-sop-template) |

## 按风险选用（原则）

| 任务 | 做法 | 验收 |
|---|---|---|
| 改写通知、普通话术 | 轻量模型 + 快检 | 语气与承诺边界 |
| 总结培训、解释概念 | 中档起步 | 结构与遗漏 |
| 多资料交叉、重要方案 | 强模型 + 人工复核 | 条件与行动建议 |
| 今日价格/政策/版本 | 任意模型 + **官方核验** | 来源与日期 |
| 批量打标 | 轻量初筛 + 抽样 | 一致性 |

排行榜与价目会变，只作参考，不作对客依据。

## 使用底线

1. AI 输出不是最终决策；承诺/报价/合同/政策必须负责人确认  
2. 客户资料先脱敏  
3. 易变事实必须核验并记日期  
4. 工具有权限 ≠ 可以跳过检查  

## 三项练习（验收）

**练习 1 · 解释 Context**  
用新人能懂的话：一句话定义 + 生活类比 + 业务例子 + 2 个误区。  
合格：说清「当前可见信息 ≠ 永久记忆」。

**练习 2 · 改写指责句**  
把「你们资料太乱，AI 帮不了」改成对客户老板可说的表述（≤100 字）：不指责、讲业务价值、不夸大 AI。  

**练习 3 · 是否核验**  
对「今日平台收费规则」「解释 Token」「改写回访纪要」判断是否必须核验并说明理由。

## 培训通过标准

- [ ] 能自述 LLM / Context / Prompt / Tool / Agent / Skill  
- [ ] 能判断何时直聊 / 加资料 / 联网 / 沉淀 Skill  
- [ ] 能完成合格话术或摘要或复盘动作项  
- [ ] 知道脱敏与对客核验触发点  

## 风险边界

- 本路径是通用基础，不替代业务域专业培训  
- 行业合规（医疗、金融、出口管制等）另加专项
