# 对客内容 AI 起草 SOP：定调、生成、自检、人工发布

AI 是助理不是署名作者。**经验、事实、责任**在人；速度与变体在 AI。

## 原则

| 原则 | 含义 |
|---|---|
| EEAT 底线 | 经验、专业、权威、可信要体现在终稿 |
| 只基于给定信息 | 不确定处标 `[待确认]`，禁止编造 |
| 人工必过 | 术语、案例、承诺、法律相关句 |
| 与 SEO 目标对齐 | 主词与受众在定调阶段写死 |

独立站业务诊断见：[SEO 诊断](/practice/seo-basic-diagnosis)、[落地页](/practice/b2b-landing-page)。

## 五步法

### 1. 定调（人，约 5–10 分钟）

```text
内容类型：产品描述 / 博客 / FAQ
受众（职位与场景）：
主关键词 1–2 个：
3 个可核对卖点（能量化更好）：
差异点（相对常见方案）：
已有事实：认证 / 规格 / 产地 / 交期范围：
绝对不能写：
发布渠道与语气：
```

### 2. 生成（AI）

要求结构清晰，例如产品页：

1. 标题（含主词与特性）  
2. 价值卖点列表（价值优先于参数堆砌）  
3. 场景 / 问题—方案  
4. 参数表  
5. FAQ  
6. CTA（无胁迫、无虚假稀缺）  

指令中写明：**仅使用给定信息；缺口标待确认。**

### 3. 自检（AI，第二轮）

```text
请只做质检，不要扩写：
- 是否出现未提供的数据/案例？
- 是否空洞形容词过多？
- 主关键词是否自然出现？
- 哪些句子必须人工补经验？
列出问题清单即可。
```

### 4. 注入经验（人）

- 补真实场景与边界条件  
- 删夸大与无法举证句  
- 统一术语与单位  
- 补作者/公司可验证信息（按品牌规范）  

### 5. 发布前核验

- [ ] 事实与参数可回溯  
- [ ] 无未授权客户名与订单细节  
- [ ] CTA 与销售流程一致  
- [ ] 过 [对核验清单](/practice/ai-verification-checklist) 的对客项  

## 博客 / FAQ 差异

| 类型 | 定调重点 | AI 易翻车点 |
|---|---|---|
| 博客 | 搜索意图、大纲、经验角度 | 编造统计与「研究显示」 |
| FAQ | 真实买家问题列表 | 回答越权承诺 |

## 完成标准

- 定调表可归档，后人能重做同题
- 终稿能指出「哪一段是人注入的经验」
- 无 `[待确认]` 残留上线（或有意保留为内部稿）

## 风险边界

- SEO 算法与 AI 概览展示会变；内容质量原则优先于技巧投机
- 翻译稿需母语或专业审校，尤其是安全与合规句
