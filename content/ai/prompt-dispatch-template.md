# AI 派工模板：把对客任务说清楚的最小字段

AI 输出差，多半不是模型「不够聪明」，而是派工单缺字段。

## 通用骨架（复制即用）

```text
你现在扮演：
我要完成的任务：
使用对象是谁：
业务背景是：
我提供的资料是：
输出格式要求：
不能做什么（禁区）：
哪些信息必须标注「不确定 / 待核验」：
完成标准（怎样算可用）：
```

## 字段说明

| 字段 | 写什么 | 反例 |
|---|---|---|
| 扮演 | 能力边界清晰的角色 | 「你是全能专家」 |
| 任务 | 可验收的单一目标 | 「帮我弄一下客户」 |
| 对象 | 读者/客户角色与阶段 | 不写读者 |
| 背景 | 行业、目标、约束 | 假设模型知道你的项目 |
| 资料 | 已脱敏材料范围 | 粘贴未授权隐私 |
| 格式 | 标题、表格、字数、语言 | 「随便写」 |
| 禁区 | 不可承诺、不可编造 | 不写边界 |
| 待核验 | 事实、价格、政策、版本 | 默认全真 |
| 完成标准 | 可直接用 / 仅草稿 | 无验收 |

## 场景补丁

| 场景 | 必须加的信息 | 合格输出 |
|---|---|---|
| 客户话术 | 阶段、语气、承诺边界 | 无越权承诺，可复制 |
| 资料总结 | 范围、层级、要回答的问题 | 写清覆盖与未覆盖 |
| 概念讲解 | 听众、类比、误区 | 新人能复述 |
| 复盘纪要 | 事实 / 问题 / 原因 / 动作 / 责任人 | 有下次检查点 |
| 培训材料 | 时长、练习、验收 | 可直接开课 |
| 询盘草稿 | 买家信号、产品边界、MOQ/交期假设 | 标出待业务确认项 |

## 外贸询盘草稿示例（结构）

```text
你现在扮演：B2B 外贸业务助理（不替代业务员做承诺）
任务：根据下列询盘写一封英文首回草稿
对象：海外采购联系人，首次接触
背景：[产品类目]；我们可做 [范围]；不可做 [范围]
资料：询盘原文（已脱敏）+ 产品一页纸要点
格式：邮件；120–180 词；3 段：确认需求 / 匹配点 / 3 个澄清问题
禁区：不报未授权价格；不编造认证与交期；不贬低竞品
待核验：任何数字、交期、认证一律标注 [待确认]
完成标准：业务员改 3 分钟内可发出
```

## 少样本（可选增强）

固定格式任务，补 2–4 个「输入 → 输出」例子，比只写形容词更稳。

## 完成标准

- [ ] 任意同事只看派工单能重做任务
- [ ] 禁区与待核验非空
- [ ] 输出默认先当草稿，再走 [核验清单](/practice/ai-verification-checklist)

## 风险边界

- 模板不能替代脱敏与权限
- 对客发送前必须人工过目
