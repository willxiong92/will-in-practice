# 国际站核心指标一页纸：从询盘有效到订单质量

## 场景与问题

**读者**：运营负责人、团队主管、做陪跑复盘的 FDE。

**场景**：周会只报曝光；广告花费上升但说不清业务结果；CRM 与平台数据对不上。

**痛点**：指标过多无法行动；有效询盘无定义；订单质量被件数掩盖。

## 核心判断

先固定 **少数结果链指标**，再讨论流量与创意：

```text
有效询盘 → 可跟进客户 → 商机 → 订单质量（均价/结构）
```

没有「有效」定义时，所有转化率都不可比。  
具体字段名因 CRM/平台而异；下表是**推荐公开口径**（有效数/询盘数），不是某一后台固定字段，也不是内部同步专用公式。团队必须写成自己的定义表并标注版本号。

## 方法拆解

### 推荐五类指标（可裁剪）

| 指标 | 要回答的问题 | 口径提示 |
|---|---|---|
| 询盘有效率 | 进来的对话有多少值得跟 | 有效数 / 同期询盘数；「有效」须书面定义 |
| 客户激活率 | 潜客有多少进入可经营状态 | 进入意向阶段数 / 潜客池（阶段名自定） |
| 商机率 | 意向客户有多少形成明确商机 | 商机数 / 意向客户数 |
| 订单均价 | 成交结构是否变差 | 周期成交额 / 成交单数 |
| 毛利或贡献结构 | 是否在做亏本热闹 | 用你方可信的毛利/贡献口径；没有则先不报 |

### 统计纪律

1. 固定观察窗口（如自然月或近 28 天）  
2. 平台侧与 CRM 侧分别记账，对不上时先修同步再争结论  
3. 推广花费与有效询盘、商机同表呈现  
4. 禁止用不可追溯的截图口播代替指标  

### 数据断层时怎么报

| 情况 | 做法 |
|---|---|
| 询盘进 CRM 丢失 | 标「采集不可信」，优先修链路 |
| 只能看到订单后半段 | 交易结论受限，不推断全漏斗 |
| 产品重复导致统计乱 | 先定「主推 SKU 映射」，暂缓产品维分析 |

（系统对接细节因工具而异，公开稿不绑定内部 CRM 产品名。）

## 风险边界

**适用**：有基本记录习惯的团队。

**不适用**：把示例指标当平台官方 KPI；无财务口径时强行报「毛利率真理」。

**常见错误**：指标 30 个；有效询盘定义每人不同；用曝光掩盖无商机。

### 验收标准

- 团队对「有效询盘」定义一致且书面化  
- 连续两个月能填同一张卡  
- 周会至少回顾有效询盘与商机，而非只回顾曝光
