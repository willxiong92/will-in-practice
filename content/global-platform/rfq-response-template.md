# 国际站 RFQ 回复模板：结构、优先级与额度分配

RFQ 拼的是「匹配度 × 速度 × 可执行报价」。模板解决速度；优先级解决把时间花在刀刃上。

## 适合谁

- 每日 RFQ 多、回复质量不稳定
- 有报价次数/额度限制的团队
- 希望业务与运营共用同一回复骨架

## 每日操作顺序

1. 按**产品匹配度**排序，不按发布时间盲回  
2. 高匹配优先；明显不匹配果断跳过  
3. 额度/次数有限时，先保证主推品与目标市场  
4. 回复后记入跟进：是否继续 IM / 邮件 / 样品  

与 [询盘分级与背调](/practice/inquiry-rfq-playbook) 联用。

## 回复骨架（可复制改写）

**1. 确认需求**

- 复述：品名/规格/数量区间/目标市场（用买家原词 + 你的理解）
- 一句澄清：若有缺失字段（材质、认证、交期）

**2. 匹配说明**

- 你能做的型号/方案（1–2 个，勿倾倒目录）
- 与需求的对齐点：规格、场景、交期能力

**3. 商务要点（可执行）**

- 参考价逻辑或区间（按公司授权）
- MOQ、交期、包装、付款方式（常规）
- 样品是否可提供及条件

**4. 证据与信任**

- 可公开的认证/测试/产能说明（有则写，无则不编）
- 下一步：要图纸确认 / 视频验厂 / 即时通讯细谈

**5. 明确 CTA**

- 请对方确认 2–3 个关键变量（规格、数量、交期）
- 给出你的可联系时段或站内通道

## 字段检查（发出前）

- [ ] 数量单位与买家一致  
- [ ] 交期是可兑现的工作日/自然日口径  
- [ ] 价格假设写清（EXW/FOB 等，若已报价）  
- [ ] 无禁限售与夸大承诺词  
- [ ] 附件/链接可打开且与型号一致  

## 优先级简易表

| 信号 | 处理 |
|---|---|
| 规格具体 + 数量合理 + 市场匹配 | 高优先，完整骨架回复 |
| 规格模糊但品类主推 | 中优先，先澄清再报价 |
| 数量极端 / 明显询证/同行口吻 | 低优先或标准短回复 |
| 完全不在能力范围 | 不消耗额度；可礼貌说明不可做 |

## 跟进节奏（回复后）

| 节点 | 动作 |
|---|---|
| 当日 | 确认是否已读/是否转即时沟通 |
| 1–3 天 | 补充资料或样品方案 |
| 1 周 | 无反馈则价值补充一次，不连续催 |

## 完成标准

- 团队 RFQ 回复能套进同一骨架抽查
- 主推品消耗的额度占比可解释
- 高匹配 RFQ 有跟进记录，不是「回完即忘」

## 风险边界

- 各平台 RFQ 权益、次数与排序规则会变，以当期后台为准
- 不引用无法复核的「提升成交率 xx%」承诺
- 价格与交期须经内部授权，模板不能变成乱报价工具
