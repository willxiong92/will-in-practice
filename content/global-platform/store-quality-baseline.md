# 国际站店铺质量底座：先过线再谈流量倾斜

## 场景与问题

**读者**：国际站运营负责人、主管、陪跑 FDE。

**场景**：想冲等级/要更多免费流量；付费效果差；新店冷启动。

**痛点**：只记平台口号指标，不修服务与内容；底座未过线就加预算。

## 核心判断

本文是 [运营框架](/practice/platform-ops-framework) 中 **「店铺质量 / 底座优先于投放」** 的验收展开，不是另一套总图。

多数 B2B 平台会把 **服务质量、响应、交易与内容完整度** 纳入流量与曝光相关机制（具体规则以当期官方说明为准）。

公开可执行的判断是：

```text
合规账户 → 响应与对话质量 → 可核验的交易/履约信号 → 主推内容完整
  → 再谈等级冲刺与放量
```

**不在此文写死**「几星等于多少倍买家」等易变且难独立核验的宣传数字。

## 方法拆解

### 四块底座

| 块 | 核心问题 | 检查方向 |
|---|---|---|
| 账户与合规 | 能否稳定经营 | 公司信息、类目、禁限售、认证材料 |
| 响应与服务 | 买家愿不愿继续聊 | 首回时效、完结率、差评/纠纷处理 |
| 交易信号 | 平台是否看到履约能力 | 订单完成、取消/纠纷率（以你可见数据为准） |
| 内容供给 | 买家能否快速理解供给 | 主推品标题主图商详、关键词与场景 |

### 冷启动建议顺序

1. 合规与主推类目正确  
2. 主推品内容过 [发品清单](/practice/listing-content-checklist)  
3. 响应节奏过 [日操](/practice/platform-daily-ops)  
4. 小预算验证 + 漏斗分层（见 [曝光到询盘](/practice/exposure-inquiry-funnel)，同批草稿，审核中）  
5. 再谈会员/等级路径（以官方权益页为准，本文不列产品名对照表）  

### 经营模式选择（原则）

按能力选主战场，可并行但需优先级：

| 方向 | 更适合 |
|---|---|
| 定制/打样强 | 强调工程能力与交期管理 |
| 现货/快反 | 强调库存与发货确定性 |
| 产能/验厂 | 强调工厂资质与稳定性 |

新团队建议先做深一条，再扩展。

## 风险边界

**适用**：需要统一「能不能放量」的内部验收。

**不适用**：替代平台官方星级/会员规则手册；用过时截图指导冲级。

**常见错误**：只刷行为指标；内容空心；忽略纠纷与履约。

## 行动工具

### 工具：底座验收清单

**账户与合规**

- [ ] 公司与联系信息完整可核验  
- [ ] 主类目与真实供给一致  
- [ ] 上架前过 [合规基础](/practice/compliance-basics)  

**响应与服务**

- [ ] 有书面首回 SLA  
- [ ] 近 7 日抽检 ≥5 条对话：无长期买家末条悬挂  
- [ ] 站内信/询盘池无「无人认领」超过 SLA 的条目  
- [ ] 投诉/纠纷有升级路径与时限  

**交易与内容**

- [ ] 主推品列表 ≤10 且可从搜索/类目找到  
- [ ] 每个主推品商详可见：核心参数、MOQ/起订逻辑、交期说明、询盘入口  
- [ ] 关键交易状态在内部系统可追踪  
- [ ] 近 28 日主推品至少有一次内容或数据回顾记录  

### 验收标准

- 团队能回答「当前能不能加预算」并指出未过线的那一块  
- 底座项有检查日期与负责人  
- 放量前漏斗层结论与底座结论不互相矛盾
