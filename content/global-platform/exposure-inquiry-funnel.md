# 国际站曝光到询盘漏斗：卡住哪一层先修哪一层

## 场景与问题

**读者**：国际站运营、外贸销售、诊断店铺的 FDE。

**场景**：老板说「没询盘」；广告在烧；产品很多但转化看不清。

**痛点**：把所有问题都归因于「要加预算」或「平台没流量」。

## 核心判断

先分层，再动作：

```text
曝光 → 点击/进店 → 商详理解 → 发起询盘 → 有效首回与跟进
```

**与 [推广诊断](/practice/ads-diagnosis-checklist) 的分工（必读）**

| 本文 | 推广诊断 |
|---|---|
| **全店/主推品** 自然+付费合计的曝光→询盘→首回 | **付费账户**结构、出价、搜索词浪费、投放复盘 |
| 判断「卡在哪一层」 | 判断「预算怎么花、词与素材怎么改」 |
| 输出给投放一句话方向 | 输出 P0/P1 投放动作 |

若已判定卡在投放结构/出价/否词，**不要在本文展开**，直接转推广诊断。

| 卡点层 | 优先动作方向 |
|---|---|
| 曝光不足 | 主推品、类目/关键词匹配、店铺质量；确认非纯预算问题后再放量 |
| 有曝光无进店 | 主图标题、价格信息、信任标签（素材层） |
| 有进店无询盘 | 商详参数、MOQ/交期、CTA、聊天入口 |
| 有询盘无效 | 筛选与话术、买家分层、产品匹配 |
| 有询盘无推进 | 响应节奏、方案/样品、内部协同 → [日操](/practice/platform-daily-ops) / [商机](/practice/inquiry-rfq-playbook) |

指标取自 **B2B 平台后台 + 对话质量**，不是独立站 GA 一套。

## 方法拆解

### 1. 选观察窗口与主推品

- 窗口：近 7/28 天（团队固定一种）  
- 只诊断 **主推 3–10 个品**，全店平均会稀释问题  

### 2. 逐层取证（有什么用什么）

| 层 | 尽量收集的证据 |
|---|---|
| 曝光 | 平台展示/曝光类指标（名称因后台而异） |
| 进店/点击 | 点击率、访客 |
| 商详 | 停留、关键页访问（若有） |
| 询盘 | 询盘数、RFQ 数 |
| 有效与推进 | CRM/表格中的有效、样品、报价节点 |

缺数据时：**能定性就定性**，并写「数据缺口」，不要编数字。

### 3. 分层结论句（交给投放/内容的一句话）

- 「曝光不足，先修主推与底座，暂缓加预算」  
- 「曝光有、进店差，先改主图标题」  
- 「进店有、询盘差，先改商详与入口」  
- 「询盘有、推进差，先修响应与方案，不归咎投放」  
- 「仅付费侧异常」→ 转 [推广诊断](/practice/ads-diagnosis-checklist)

## 风险边界

**适用**：有基础后台数据或可导出报表。

**不适用**：承诺「做完漏斗诊断一定涨询盘」；把无法核验的行业转化率当标准。

**常见错误**：全店一起算；只看花费；忽略响应层。

### 验收标准

- 结论落在明确一层，而非「再加点预算试试」  
- 主推品清单书面存在  
- 动作 ≤3 且有 Owner
