> 一句话：7 大 Skill 群（选品/品牌/发品/营销/商机/成交/分析）+ Agent 编排。

## 定位

AWB 不是“另一个资料库”，而是把 AI 能力按客户经营任务组织起来。新人 FDE 使用 AWB 时，不是先问“有什么工具”，而是先判断客户卡在哪个经营动作，再调用对应 Skill 或 Agent。

## 7 大 Skill 群

| Skill 群 | 业务含义 | 典型能力 | 常见客户问题 |
|-|-|-|-|
| 商机洞察 · 战略选品 | 买什么、卖给谁 | AI 选品、供应链规划、市场机会判断 | 客户不知道主推什么产品、进哪个市场 |
| 品牌定义 · 视觉设计 | 品牌定位、视觉表达 | 品牌设计、商业策划、视觉物料生成 | 客户有产品但表达弱、信任感不足 |
| 智能发品 · 流量优化 | 上架效率、自然流量 | AI 生图、标题关键词、内容优化 | 客户发品慢、内容弱、自然流量差 |
| 智能营销 · 客户开发 | 获客触达、客户开发 | AI 智能营销、AI 客户开发、AIReach | 客户缺潜客名单和主动触达能力 |
| 商机管理 · 客户分层 | 商机响应、客户经营 | AI 智能接待、客户管理、客户分层 | 客户有线索但响应慢、跟进乱 |
| 成交转化 · 临门一脚 | 成交转化、提升效能 | AI 询价报价、AI 成交助手、异议处理 | 客户报价慢、谈判弱、成交推进慢 |
| 经营分析 · 风控保障 | 经营分析、风险保障 | 经营驾驶舱、风险管理、数据分析 | 管理层看不清业务问题和风险 |

## 和经营公式的关系

| 经营公式 | 优先调用的 Skill 群 | 典型任务 |
|-|-|-|
| 更多有效线索 | 智能营销 · 客户开发 / 智能发品 · 流量优化 | AIReach 自动开发、SHOP 内容和流量优化 |
| 更高转化率 | 商机管理 · 客户分层 / 成交转化 · 临门一脚 | 新客转化、商机推进、异议处理 |
| 更高客单价 | 品牌定义 · 视觉设计 / 成交转化 · 临门一脚 | 方案价值表达、组合销售、大客户经营 |
| 更高毛利率 | 经营分析 · 风控保障 / 商机洞察 · 战略选品 | 报价纪律、订单结构、经营数据分析 |

## Agent 编排方式

| Agent 类型 | 输入 | 组合能力 | 输出 |
|-|-|-|-|
| SHOP 首轮诊断 Agent | 站点、产品、目标市场、流量和询盘情况 | 流量判断、页面五要素、线索沉淀检查 | SHOP 诊断结论和 3 条优先动作 |
| AIReach 自动开发 Agent | 目标市场、买家类型、主推产品、触达数据 | 挖潜、背调、触达、孵化、入库检查 | 自动开发闭环诊断 |
| 新客转化 Agent | 询盘、客户、商机、跟进和成交数据 | 需求确认、方案推进、商机质检、复盘 | 新客户转化诊断 |
| QBR 草稿 Agent | 客户数据、阶段目标、执行动作、结果 | 数据分析、客户健康、下一阶段建议 | 阶段复盘初稿 |

## 新人使用步骤

1. 先判断客户问题属于哪个经营公式象限。
2. 再选择对应 Skill 群，不直接从工具名出发。
3. 如果是标准任务，优先调用对应 Agent。
4. 如果 Agent 输出缺证据，回到产品矩阵和业务页补输入。
5. 服务结束后，把有效 Prompt、模板、案例和判断规则沉淀回平台化能力建设。

## Bot 回答样稿

**新人提问**：AWB 是什么？我怎么知道该用哪个 AI 能力？

**建议回答**：

AWB 可以理解为经营大脑，不是单个工具。你先判断客户卡在哪个经营动作，再调用对应 Skill 群。

如果客户缺线索，优先看智能营销、客户开发和流量优化；如果客户有线索但成交少，看商机管理和成交转化；如果客户想做大客户或提升方案价值，看品牌定义、视觉设计和成交转化；如果客户要提升毛利和经营效率，看经营分析、风控保障和战略选品。

标准任务优先用 Agent，比如 SHOP 首轮诊断、AIReach 自动开发、新客户转化或 QBR 草稿。输出后还要沉淀模板、Prompt、案例和规则，不能只完成一次性问答。
