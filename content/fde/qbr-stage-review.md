# QBR 不是贴数据：阶段复盘的标准结构

阶段复盘要证明：本阶段做了什么、带来什么变化、还卡在哪里、下一阶段怎么推进。

## 适用场景

- 完成 2 周或 1 个月陪跑，需要阶段总结
- 客户问“服务有没有效果”
- 准备季度经营回顾
- 需要从结果倒推下一轮动作

## 复盘六段

| 模块 | 要回答的问题 |
|---|---|
| 背景目标 | 当初为什么做？目标是什么？ |
| 执行动作 | 实际做了哪些？谁做的？ |
| 数据变化 | 哪些指标变了，哪些没变？ |
| 业务判断 | 变化说明什么？卡点在哪？ |
| 下一步 | 下阶段主攻什么？ |
| 沉淀资产 | 哪些经验可复用？ |

## 指标怎么选

| 方向 | 可复盘指标示例 |
|---|---|
| 线索 | 有效流量、询盘数、线索有效率、转客户率 |
| 转化 | 建档率、首响、商机推进、成交率、沉睡激活 |
| 执行保障 | 任务完成率、责任人检查、过程记录 |

原则：先说明目标口径，再放数字；过程量必须连接业务结果。

## 常见错误

| 错误 | 应改成 |
|---|---|
| 只贴截图 | 截图后必须有判断和下一步 |
| 只报过程量 | 连接到目标和结果 |
| 只讲做了很多 | 只保留最关键的 1-3 个动作 |
| 只讲客户没执行 | 拆成能力、制度、责任、优先级 |
| 下一步太多 | 只保留一个主攻方向和 3 个动作 |

## 完成标准

- 客户能听懂结论，而不只是看到数据表
- 有明确下一阶段主攻
- 至少沉淀 1 个模板、话术或案例

## 风险边界

阶段性经验不等于行业通论。指标改善可能受季节、市场和客户投入影响，复盘中要写清假设。
