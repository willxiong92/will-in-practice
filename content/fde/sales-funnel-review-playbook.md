# 销售漏斗与团队复盘：用过程而不是只看成交

销售 = **素材准备 + 技巧询盘/跟进 + 复盘优化**。没有过程表，周会只能复盘情绪。

## 阶段漏斗（可按行业改名）

| 阶段 | 业务含义（外贸示例） | 看什么 |
|---|---|---|
| S0 | 首次有效触达 / 首回 | 响应质量与时效 |
| S1 | 需求与款式/方案讨论 | 是否进入实质对话 |
| S2 | 同意样品 / 方案深化 | 意愿是否升级 |
| S3 | 样品/方案通过 | 技术与预期对齐 |
| S4 | 成交 / 签约节点 | 商务闭环 |

阶段名不重要，**可统计、可复盘** 才重要。

## 如何读转化

1. 固定观察窗口（如近 30/90 天）  
2. 按人 / 按品类 / 按市场算 S0→S1… 转化  
3. 与团队中位对比，找「前强后弱 / 前弱后强」  

**诊断逻辑（示例）：**

- 首回强、样品弱 → 查样品节奏与规格确认（见 [样品清单](/practice/sample-process-checklist)）  
- 讨论强、首回弱 → 查响应模板与在线覆盖  
- 成交强、前端弱 → 线索质量与获客结构  

> 任何「行业平均转化率」数字都会过时且不可外推。用**你团队自己的基线**，不要抄别人的百分比当 KPI。

## 复盘会六步

1. **交案例**：完整沟通记录（可脱敏）  
2. **预分析**：指定人会前读完并标疑问  
3. **自述**：提交人讲做了什么、为何、结果  
4. **点评**：指出盲点，不人身攻击  
5. **共定动作**：可检查的下一步 + 责任人 + 日期  
6. **回看改善**：下次会先看上次动作有无落地  

### 频次与样本量

- 必须循环开，间隔企业自定（常见周/双周）  
- 冷启动可多看存量线索，形成规律后再聚焦在途单  
- 依据要可证实；无过程无结果的个案单独处理  

## 周会最小产出

- [ ] 每人 1 个卡点阶段  
- [ ] 团队 Top 1 共性问题  
- [ ] ≤3 条改进动作（可验收）  
- [ ] 需升级的风险客户名单  

## 完成标准

- 漏斗阶段定义书面统一  
- 复盘记录可检索，不是聊天消散  
- 个人改进能量化到阶段转化，而不是「更努力」  

## 风险边界

- 复盘禁止当众羞辱；问题对事不对人  
- 沟通记录含客户隐私，按公司规范脱敏存档
