# 更多有效线索：被动承接、主动开发与线索入库

目标不是线索数量更大，而是 **资源池更大 × 线索质量更高**。

## 交接口径

- 本象限负责：`线索 -> 客户`
- 客户之后的商机与成交：交给「更高转化率」

## 四条路径

| 模块 | 解决什么 | 先看什么 |
|---|---|---|
| 有效流量 | 有没有合适买家进来 | 市场、意图、产品匹配 |
| 询盘转化 | 进来后有没有留下询盘 | 首屏、信任、CTA、表单 |
| 主动开发 | 有没有主动找到并触达潜客 | 画像、触达、孵化 |
| 线索沉淀 | 线索有没有变成可经营客户 | 归集、去重、有效性、分配 |

## 问题分流

| 客户问题 | 先看 |
|---|---|
| 独立站没询盘 | 有效流量 vs 页面转化 |
| 主动开发没效果 | 挖潜、触达、孵化、入库是否连续 |
| 有线索但销售接不住 | 线索沉淀与转客户 |
| 不知道先做站还是先开发 | 被动先看承接阵地，主动先看开发闭环；交接乱先看沉淀 |

## 完成标准

- 能指出客户主卡在四条路径中的哪一条
- 输出不超过 3 条优先动作
- 有效线索最终进入统一客户资产，而不是散落在个人聊天
