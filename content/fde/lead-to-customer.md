> **TL;DR**：更多有效线索不是“有询盘就结束”。FDE 要确保线索被统一归集、去重、判定有效性、转成客户并分配跟进，否则 SHOP 和 AIReach 的获客动作不会变成客户资产。

## 适用场景

| 场景 | 典型表现 | 本页能解决什么 |
|-|-|-|
| SHOP / AIReach 已产生线索 | 有表单、回复、名片或名单，但后续不清楚 | 让新人判断线索是否进入客户资产池 |
| 客户渠道很多 | 独立站、邮件、社媒、展会、老资料分散 | 建立统一归集和去重口径 |
| 线索质量争议大 | 销售觉得线索不准，运营觉得已产生线索 | 用有效线索标准减少扯皮 |
| FDE 要交接给转化链路 | 已有线索，但还不是商机 | 明确“线索 -> 客户”的交接口径 |

## 交接口径

| 层级 | 本页负责 | 不负责 |
|-|-|-|
| 线索 | 识别来源、去重、判定有效性 | 直接承诺成交 |
| 客户 | 建档、补全画像、分配责任人、记录下一步动作 | 商机阶段推进 |
| 商机 | 只标记是否具备商机创建条件 | 询盘、样品、报价、成交 SOP |

交接原则：

- “更多有效线索”只负责到 `线索 -> 客户`。
- `客户 -> 商机 -> 成交` 交给“更高转化率”。
- 没有进入 CRM 的线索，不算客户资产。
- 没有来源、质量、责任人和下一步动作的客户，不算可运营客户。

## 标准闭环

```text
线索产生
  -> 统一归集
  -> 去重与合并
  -> 有效性判断
  -> 客户建档与画像补全
  -> 分配责任人与跟进动作
  -> 交给更高转化率继续推进
```

## 第一步：统一归集线索

| 来源 | 常见线索形态 | 新人检查点 |
|-|-|-|
| SHOP | 表单询盘、在线咨询、下载资料、留言 | 是否记录来源页面、产品、市场和需求 |
| AIReach | 回复、打开点击、高潜客户、待激活客户 | 是否记录触达轮次、背调结果和回复意图 |
| 社媒 / 展会 | 名片、私信、群聊、活动名单 | 是否记录来源场景、联系人和初步需求 |
| 老资料 | 历史名单、沉睡客户、未分配客户 | 是否判重、补全画像并重新激活 |

## 第二步：线索去重与合并

| 检查项 | 判断标准 | 常见问题 | 标准动作 |
|-|-|-|-|
| 公司维度 | 公司名称、官网、邮箱域名、国家是否相同 | 同一家公司多次建档 | 合并到同一客户 |
| 联系人维度 | 姓名、邮箱、WhatsApp、LinkedIn 是否重复 | 一个联系人分散在多个记录 | 合并联系人与沟通记录 |
| 来源维度 | 是否保留首次来源和最近来源 | 只看最新来源，丢失获客归因 | 保留首触来源、最近来源、有效来源 |
| 责任人维度 | 是否明确跟进人和协作人 | 线索进库后没人负责 | 分配责任人和下一步动作 |

## 第三步：判断有效线索

| 维度 | 有效信号 | 无效或低优先信号 |
|-|-|-|
| 需求相关 | 询问产品、价格、定制、交期、认证、样品 | 泛泛咨询、招聘、广告、供应商推销 |
| 客户匹配 | 国家、行业、买家类型、采购场景匹配 | 非目标市场、非目标客户类型 |
| 联系可达 | 有邮箱、电话、社媒或官网可追溯 | 联系方式缺失或明显虚假 |
| 推进价值 | 有预算、时间、采购意向或复购可能 | 只浏览、无回应、无业务相关性 |

有效线索最低标准：

1. 能识别客户主体。
2. 能确认业务相关性。
3. 能找到可触达联系人或渠道。
4. 能写清下一步跟进动作。

## 第四步：转成客户资产

| 客户字段 | 最低要求 | 为什么重要 |
|-|-|-|
| 客户名称 | 公司名或可识别主体 | 避免线索只停留在聊天或表单里 |
| 来源 | SHOP / AIReach / 社媒 / 展会 / 其他 | 后续复盘获客质量 |
| 国家 / 地区 | 至少填到国家 | 判断市场匹配和分配策略 |
| 主推产品 | 与客户需求关联的产品 | 连接后续商机和方案 |
| 联系人 | 至少 1 个可触达人 | 保障销售跟进 |
| 线索质量 | 高 / 中 / 低 或客户自定义等级 | 决定跟进优先级 |
| 下一步动作 | 首响、背调、建联、创建商机、继续孵化 | 防止进库后停滞 |

## FDE 标准动作

1. 明确客户当前所有线索来源，不只看一个渠道。
2. 检查线索是否进入统一客户池。
3. 按公司、联系人、来源做去重与合并。
4. 用有效线索标准判断质量，不把所有名单都当作有效线索。
5. 把有效线索转成客户，并补齐来源、国家、产品、联系人、质量和下一步动作。
6. 对达到商机条件的客户，交给“更高转化率”继续推进。

## Bot 回答样稿

**新人提问**：SHOP 和 AIReach 都有线索了，下一步是不是直接让销售跟？

**建议回答**：

不要直接丢给销售。先确认线索有没有变成客户资产。

你需要做四步：第一，把 SHOP、AIReach、社媒、展会等来源统一归集；第二，按公司、联系人和来源做去重合并；第三，用需求相关、客户匹配、联系可达、推进价值判断是否是有效线索；第四，把有效线索转成 CRM 客户，并补齐来源、国家、产品、联系人、线索质量和下一步动作。

如果客户已经具备明确需求、联系人和推进动作，再交给“更高转化率”链路创建商机并推进成交。

## 四验收

| 验收 | 标准 |
|-|-|
| 认知 | 新人能说清“线索不等于客户资产” |
| 判断 | 能判断问题在未归集、未去重、质量不清还是入库后停滞 |
| 执行 | 能按模板完成一次线索来源盘点和客户资产补齐 |
| 沉淀 | 能把线索来源、有效率和转客户结果回写为复盘资产 |
