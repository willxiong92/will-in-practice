# 顾问式客户沟通：委托、倾听、界定、构想、承诺

顾问式不是「更会讲」，而是先获得**建议权**，再一起定义问题与代价。

## 三种模式对照

| 模式 | 表现 | 风险 |
|---|---|---|
| 客服式 | 有问才答 | 停在救火，推不动改变 |
| 推销式 | 很快甩方案 | 客户抵触、缺少参与 |
| 顾问式 | 理解→界定→共创 | 开口慢，落地深 |

## 五段流程

### 1. 赢得委托

目标：客户愿意说真话、给时间。  
信号：主动补充背景、接受议程、不急着砍价式打断。

### 2. 倾听

- 让故事讲完整  
- 复述确认事实与情绪  
- 客户过早要方案时，克制「立刻给答案」  

倾听是在赢「继续参与」的权利。

### 3. 界定问题

客户说的常是症状。要做：

- 模糊 → 可描述的业务问题  
- 暴露制约、隐藏代价、关键假设  
- 与客户对齐「我们到底在解什么」  

### 4. 构想未来

在给方案前，先把「更好的状态」说具体：谁的工作变了、什么指标变了、大概路径。  
不是画大饼，是让行动值得发生。

### 5. 共同承诺

「好的」不够。要落到：

- 双方各做什么  
- 时间与检查点  
- 不做会怎样（代价）  

可与 [陪跑计划](/practice/customer-rollout-plan) 衔接。

## 现场检查清单

- [ ] 我是否已有建议权，还是仍在推销？  
- [ ] 问题是客户原话，还是我们共同界定的？  
- [ ] 下一步是否双边承诺？  
- [ ] 有无越权承诺产品/效果？  

## 完成标准

- 纪要里能看到五段中的至少「界定 + 承诺」  
- 客户能复述「我们要解的问题」  
- 内部动作可进入执行诊断（见 [执行卡点](/practice/customer-execution-diagnosis)）  

## 风险边界

- 顾问式不等于替代客户做决策  
- 书籍框架需结合你的行业合规与权限
