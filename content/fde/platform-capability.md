> **TL;DR**：平台化能力建设不是资料库，而是服务组织的后方智脑。FDE 每服务一个客户，都要把有效流程、Skill、Agent 调度和案例带回平台，抽象成通用资产，再由 AI 反哺下一位 FDE / CSM。

## 定位

| 问题 | 平台化能力建设的回答 |
|-|-|
| 为什么团队知识库不能只是资料库？ | 因为资料只会存放信息，平台要把服务动作、判断标准和复盘资产转成可复用能力 |
| 为什么要从一线服务回收资产？ | 因为 FDE 的经验如果只留在个人手里，组织无法复制，也无法降低新人胜任成本 |
| 为什么要接 AI？ | 因为流程、Skill、案例和模板被结构化后，AI 才能在新人 onboarding、客户诊断、复盘沉淀中调用 |

## 平台化三性

| 三性 | 含义 | 验收方式 |
|-|-|-|
| 可视化 | 服务流程与资产看得见、可检索 | 是否有清晰目录、标签、入口和示例 |
| 可量化 | 复用率与服务复利可度量 | 是否能统计流程复用、Skill 调用、资产入库和新人胜任 |
| 可升级 | 资产随每次服务持续进化 | 是否能把新问题、新案例、新模板回流到团队库 |

## 平台闭环

```text
一线服务客户
  -> 回收流程 / Skill / Agent / 案例 / 模板
  -> 抽象成通用资产并打标签
  -> AI 调用推荐反哺下一位 FDE / CSM
  -> 新结果继续沉淀
```

这个闭环的目标不是“把文件放进库里”，而是让团队每服务一次客户，就让下一次服务更快、更准、更可复制。

## P1 流程平台化

> 把 FDE 服务从个人手艺，变成可复用的分级流程。

| 层级 | 内容 | 示例 |
|-|-|-|
| 一级流程 | 客户盘点、经营诊断、产品匹配、复盘增续 | 新客户启动、客户经营复盘 |
| 二级流程 | 线索增长、新客转化、沉睡盘活、合作客户复购 | SHOP 询盘转化诊断、AIReach 自动开发 |
| 三级流程 | 背调、触达、回访、提醒、报价后跟进 | 首轮问诊、页面诊断、线索入库、QBR 初稿 |

验收指标：

| 指标 | 判断标准 |
|-|-|
| 流程覆盖率 | 高频服务场景是否都有流程入口 |
| 流程标准化率 | 是否有输入、步骤、输出、异常和验收 |
| 流程复用率 | 新人或其他 FDE 是否能按流程完成动作 |

## P2 能力封装平台化

> 把高频判断和动作封装成 Skill，把多 Skill 编排成 Agent。

| 资产 | 说明 | 示例 |
|-|-|-|
| Skill | 可独立调用的高频能力 | 客户背调、话术生成、商机质检、复盘摘要 |
| Agent | 多个 Skill 组合成任务流 | SHOP 首轮诊断 Agent、AIReach 自动开发 Agent、QBR 草稿 Agent |
| 质检规则 | 判断 Skill / Agent 是否能上架 | 输出是否准确、完整、可执行、可复用 |

验收指标：

| 指标 | 判断标准 |
|-|-|
| Skill 数量与调用率 | 高频动作是否被封装，是否被一线真实调用 |
| Agent 场景覆盖 | 是否覆盖新人 onboarding 和高频客户服务任务 |
| 质检通过率 | 是否通过认知、判断、执行、沉淀四验收 |

## P3 资产沉淀平台化

> 服务结果必须沉淀为结构化、可检索、带标签的组织资产。

| 资产类型 | 内容 | 入库要求 |
|-|-|-|
| 案例库 | 成功案例、失败案例、客户证言 | 必须脱敏，说明场景、动作、结果和复盘 |
| 模板库 | 问诊表、话术、看板、验收单 | 必须能直接复制使用 |
| 标签编码 | 行业、产品、场景、指标、结果 | 必须能支撑检索和 AI 调用 |
| 事实资料 | 产品文档、平台规则、行业数据 | 易变事实必须标注确认时间和核验状态 |

验收指标：

| 指标 | 判断标准 |
|-|-|
| 资产入库率 | 每次重点服务后是否回收 1 个可复用资产 |
| 标签完整率 | 是否有行业、产品、场景、指标、结果标签 |
| 检索命中率 | 新人能否按问题找到资产 |
| 可见性合规率 | 团队可见内容是否完成安全和适用性审核 |

## P4 复利反哺平台化

> 平台资产不只是存档，而要进入下一次服务动作。

| 反哺对象 | 反哺方式 | 预期结果 |
|-|-|-|
| 新人 FDE | onboarding 问答、foundation 页、标准模板 | 缩短新人胜任周期 |
| 一线 FDE / CSM | 服务流程、诊断模板、案例参考 | 降低重复劳动，提高诊断一致性 |
| 主管 | 质检标准、复盘看板、资产沉淀情况 | 看见团队能力缺口和复利效果 |
| AI Bot | 读取已审核知识子集并带来源回答 | 问答可追溯，错误可回流 |

验收指标：

| 指标 | 判断标准 |
|-|-|
| 资产复用率 | 团队是否在真实服务中复用页面、模板、Skill |
| 新人胜任周期 | 新人完成标准任务所需时间是否缩短 |
| AI 推荐采纳率 | bot 推荐的流程或模板是否被使用 |
| 反馈回流率 | 被追问、看不懂、用不上、事实错误是否回写 |

## 新人怎么用

| 新人问题 | 应进入哪里 | 输出 |
|-|-|-|
| 服务完客户后，我要沉淀什么？ | P3 资产沉淀平台化 | 案例、模板、标签或 Skill 草稿 |
| 我怎么把一个好动作变成团队可复用流程？ | P1 流程平台化 | 一级 / 二级 / 三级流程 |
| 我发现一个高频操作，怎么交给 AI？ | P2 能力封装平台化 | Skill 或 Agent 编排说明 |
| 我怎么知道团队库有没有产生价值？ | P4 复利反哺平台化 | 复用率、新人胜任、采纳率、反馈回流 |

## 四验收

| 验收 | 标准 |
|-|-|
| 认知 | 新人能讲清平台化不是资料库，而是服务复利机制 |
| 判断 | 能判断一次服务结果应该沉淀为流程、Skill、Agent、案例还是模板 |
| 执行 | 能按模板完成一次服务资产沉淀 |
| 沉淀 | 能把沉淀内容回流到团队 FDE-KB，并标注可见性 |
