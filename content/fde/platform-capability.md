# FDE 平台化能力建设：让每次服务变成下一次复利

平台化不是资料库搬家，而是服务组织的后方智脑：每次服务都回收流程、Skill、案例和模板，再反哺下一位同学。

## 平台化三性

| 三性 | 含义 | 怎么验收 |
|---|---|---|
| 可视化 | 流程与资产看得见 | 有目录、标签、入口和示例 |
| 可量化 | 复用能被统计 | 有复用率、调用率、入库率 |
| 可升级 | 资产能进化 | 有反馈回流和更新节奏 |

## 平台闭环

```text
一线服务
  -> 回收流程 / Skill / 案例 / 模板
  -> 抽象为通用资产并打标签
  -> 推荐给下一次服务
  -> 新结果继续沉淀
```

## 四层建设

### P1 流程平台化

把个人手艺变成分级流程：

- 一级：客户盘点、经营诊断、复盘增续
- 二级：线索增长、新客转化、沉睡盘活、复购
- 三级：背调、触达、问诊、入库、QBR 初稿

### P2 能力封装

- Skill：可独立调用的高频能力
- Agent：多 Skill 编排成任务流
- 质检：准确、完整、可执行、可复用

### P3 资产沉淀

| 类型 | 入库要求 |
|---|---|
| 案例 | 脱敏；写清场景、动作、结果 |
| 模板 | 可直接复制使用 |
| 标签 | 行业、场景、指标、结果可检索 |
| 事实资料 | 易变事实标注确认时间 |

### P4 复利反哺

- 新人：onboarding 与标准模板
- 一线：诊断与服务流程
- 主管：质检与能力缺口
- AI：只读已审核子集，回答带来源

## 服务后沉淀模板

```markdown
# 服务资产沉淀记录

## 基本信息
- 服务场景：
- 对应象限：
- 适用对象：

## 本次有效动作
- 做了什么：
- 为什么有效：
- 可复用条件：

## 应沉淀为
- [ ] 流程 SOP
- [ ] Skill
- [ ] Agent 任务流
- [ ] 案例
- [ ] 模板

## 边界判断
- 是否含敏感信息：
- 是否需要脱敏：
- 需要谁复核：
```

## 完成标准

- 每次重点服务至少回收 1 个可复用资产
- 新人能按问题找到流程/模板
- 反馈（看不懂、用不上、事实错误）有回流位置

## 风险边界

未脱敏的客户材料、人事与内部路线图不得进入团队共享层。
