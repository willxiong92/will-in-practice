# FDE 服务体系总图：从生意链路到经营公式

FDE 不是“帮客户点功能”，而是帮助客户看清经营卡点，并推动可执行动作。

## 一张总图

```text
客户生意链路
  -> 经营公式
  -> 能力承接（渠道 / 网站 / CRM / 自动化等）
  -> 服务动作
  -> 执行保障
  -> 复盘沉淀
```

## 客户生意链路

| 链路 | 业务问题 | 服务方要回答 |
|---|---|---|
| 引流获客 | 客户从哪里来 | 有没有足够且有效的线索来源 |
| 客户沉淀 | 线索是否留下来 | 是否形成可经营客户资产 |
| 需求跟进 | 需求是否被识别 | 关键人、节奏和商机信号是否清楚 |
| 成交转化 | 商机是否推进 | 方案、报价、谈判和推进机制是否存在 |
| 复购增购 | 合作是否持续 | 分层、复盘和增购机会是否经营 |
| 经营分析 | 结果是否可复盘 | 投入、过程、结果能否对上 |

## 经营公式四象限

| 象限 | 解决的问题 | 典型动作 |
|---|---|---|
| 更多有效线索 | 资源池更大、线索质量更高 | 独立站承接、主动开发、线索入库 |
| 更高转化率 | 线索到成交全过程提效 | 建档、首响、商机推进、沉睡盘活 |
| 更高客单价 | 方案价值与组合销售 | 重点客户经营、组合方案 |
| 更高毛利率 | 结构、报价与交付稳定 | 客户结构、报价纪律、交付质量 |

交接口径：

- 「更多有效线索」负责到 `线索 -> 客户`
- 「更高转化率」承接 `客户 -> 商机 -> 成交`
- 客单和毛利不抢跑，等真实问题拉动再深做

## 一体化服务动作

| 步骤 | 动作 | 输出 |
|---|---|---|
| 1 | 激活关键阵地和基础数据 | 资产可查 |
| 2 | 选定当前最关键经营场景 | 主攻方向 |
| 3 | 提出可量化目标 | 指标口径 |
| 4 | 开展陪跑 | 任务与节奏 |
| 5 | 复盘并沉淀 | 案例 / 模板 / SOP |

## 完成标准

- 新人能讲清：链路、公式、服务动作之间的关系
- 面对客户问题时，能先归到象限，再选工具和方法
- 不把“上了系统”误当成“经营结果改善”

## 风险边界

不同团队的产品组合不同。本页保留经营判断框架，不绑定单一厂商功能清单。具体平台规则与版本需回到当期官方说明。
