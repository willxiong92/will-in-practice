# GROW 教练式引导：目标、现状、选项与行动

## 场景与问题

**读者**：FDE / CSM、顾问、需要做 1:1 辅导的主管。

**场景**：对客诊断会、方案共创、客户不执行时的辅导、团队成员卡点 1:1。

**痛点**：

- 一上来讲功能/方案，客户没有目标感就不执行  
- 只会问「现在怎么样」，现状停留在感觉  
- 选项很多，没有行动时间和责任人  
- 简化成功能宣讲，丢掉提问习惯  

## 核心判断

GROW 是常见的教练对话结构（Goal → Reality → Options → Will）：**先对齐要去哪，再看清现在在哪，再生成选项，最后锁定行动**。

核心不是「模型四个字母」，而是：

1. 目标由对方说出（或被引导发现），不是你强加  
2. 现状要用具体事实与结构，而不是空泛感受  
3. 选项先发散再收敛，避免唯一答案压迫  
4. 没有时间点与责任人的行动 = 没有行动  

本页是**实践用法**，不构成对某一商业教练课程的转述。

## 方法拆解

### 1. 四步一览

| 步骤 | 英文 | 核心问题 | 对客 / 辅导应用 |
|---|---|---|---|
| 1 | Goal | 你要什么？ | 理清业务/个人目标；可用同行对照打开视野，但不替对方定目标 |
| 2 | Reality | 现在怎样？ | 用结构化现状（结果、过程、尝试、相关方）定位真问题 |
| 3 | Options | 有哪些选择？ | 方案与路径选项；产品/方法是选项材料，不是唯一答案 |
| 4 | Will | 接下来做什么？ | 行动 + 时间 + 责任人 + 复盘点 |

### 2. 逐步怎么问

**G · 目标**

- 这个周期你最想改变的一件事是什么？  
- 怎样算「做到了」？（可观察结果）  
- 若目标冲突（老板 vs 执行人），先澄清以谁的承诺为准  

**R · 现状**（建议用维度表，避免空问）

| 维度 | 要落到 |
|---|---|
| 结果 | 关键指标或事实结果（用对方数据，不编造） |
| 过程 | 已做尝试与节奏 |
| 卡点 | 具体卡在哪一步 |
| 相关方 | 谁影响推进（销售/运营/老板/工厂等） |

**O · 选项**

- 至少列出 2–3 个可行选项  
- 每个选项：收益、代价、依赖条件  
- 常见经营选项类型（按场景裁剪，非固定产品包）：获客、沉淀与跟进、转化推进、复盘与经营分析  

**W · 行动**

- 选什么、谁做、何时完成、何时复盘  
- 写清「本周不做什么」防止摊大饼  

### 3. 简化路径与陷阱

日常可缩短为：目标 → 现状 → 选择 → 行动。  

**陷阱**：简化版容易退化成「提问一句 + 功能宣讲」。  
**对策**：每步至少一个对方自己说出的句子，再进入下一步。

## 风险边界

**适用**：对方有一定表达意愿；问题需要对方参与才能落地。

**不适用**：

- 紧急事故指挥（先止损，再复盘辅导）  
- 对方明确只要执行清单、拒绝共创时，强行长篇教练  
- 用 GROW 包装推销，跳过 Reality 直接 Options  

**常见错误**：

- 跳过 G 直接进 O  
- 用陈述代替提问  
- R 阶段没有事实结构  
- W 没有时间与责任人  
- 编造客户经营数字当案例  

**公开安全**：本文不使用真实客户经营数字；示例一律留空由读者填入自有事实。

### 工具 B：自检清单

- [ ] 对方说出了目标或明确同意一个目标表述  
- [ ] 现状至少有 2 个可核对事实  
- [ ] 选项 ≥ 2，且未在 Reality 完成前强推唯一方案  
- [ ] 每个行动有 Owner 与时间  
- [ ] 已约定复查方式  

### 验收标准

- 会后双方能复述「目标 + 下一步 + 谁负责」  
- 行动可在约定周期内检查是否发生  
- 未出现「只讲功能、无目标、无复查」的退化形态  

**缺口标注**：母版中含产品功能映射与具体经营数字案例，公开稿已删除；落地时用你方真实、已授权或脱敏事实填写会话卡。
