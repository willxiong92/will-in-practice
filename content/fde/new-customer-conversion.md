# 新客户转化：从询盘接到成交复盘

新客转化不是“多催业务员”。先确认：询盘是否建档、需求是否清楚、方案是否推进、商机是否可追踪，最后用成交与输单复盘反哺下一轮。

## 核心公式

```text
新客户转化 ≈ 询盘转化 × 样品/方案转化 × 成交转化
```

## 闭环

```text
询盘进入
  -> 建档与首响
  -> 需求确认与匹配
  -> 样品 / 方案 / 报价
  -> 创建商机与阶段推进
  -> 成交 / 输单复盘
  -> 案例与规则沉淀
```

## 四段检查

### 1. 询盘是否接住

- 建档率
- 首响时效
- 需求字段完整度
- 来源记录

### 2. 样品 / 方案是否推进

- 需求匹配
- 方案是否发出
- 反馈是否记录
- 下一步是否明确

### 3. 商机是否可管理

| 要素 | 最低标准 |
|---|---|
| 阶段 | 询盘 → 需求 → 方案 → 谈判 → 成交/输单 |
| 金额 | 有预估区间 |
| 时间 | 有节点 |
| 责任人 | 主责清晰 |
| 跟进规则 | 有节奏，如 1/3/7 |
| 停留预警 | 卡住能被发现 |

### 4. 成交与输单是否复盘

- 为什么赢 / 为什么输
- 卡在价格、交期、产品、信任还是响应
- 沉淀案例、话术、质检规则

## 完成标准

- 能指出主卡点，而不是只说“转化率低”
- 每条动作有责任人与验收
- 输单原因开始进入团队资产，而不是只存在聊天里
