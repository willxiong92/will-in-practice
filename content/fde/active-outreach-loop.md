> **TL;DR**：AIReach 的目标不是“发更多开发信”，而是跑通 `挖潜 -> 背调 -> 千邮千面触达 -> 持续孵化 -> 有效线索入库` 的自动开发闭环。

## 适用场景

| 场景 | 典型表现 | 本页能解决什么 |
|-|-|-|
| 客户想主动开发新客户 | 有目标市场，但缺潜客名单和触达节奏 | 帮新人拆出 AIReach 的完整闭环 |
| 客户用了 AIReach 但效果不清楚 | 有触达动作，但不知道线索是否有效 | 用指标判断卡在挖潜、触达还是孵化 |
| FDE 需要解释 AIReach 价值 | 客户只理解“自动发信” | 把价值讲成“自动开发有效线索” |

## 核心公式

| 层级 | 公式 | 判断问题 |
|-|-|-|
| 总公式 | 自动开发有效线索 = 潜客挖掘精准度 × 自动触达规模 × 孵化转化率 | AIReach 是否真正产生有效线索？ |
| 挖潜 | 生态画像 + 供应链上下游图谱 + 站外潜客预测 | 找到的是不是目标客户？ |
| 触达 | 千邮千面内容 + 多轮次多通道触达 | 是否以足够规模和个性化方式触达？ |
| 孵化 | 回复识别 + 待激活跟进 + 人工接管 | 潜在线索有没有被持续推进？ |

## 标准闭环

```text
目标市场与买家画像
  -> 生态画像挖潜
  -> 自动背调与画像补全
  -> 千邮千面触达
  -> 持续孵化待激活客户
  -> 有效线索进入 OKKI CRM
  -> 交给更高转化率链路继续推进
```

## 第一步：定义目标客户

| 检查项 | 新人要问的问题 | 常见问题 | 下一步动作 |
|-|-|-|-|
| 目标市场 | 客户想优先开发哪些国家或地区？ | 市场太泛，没有优先级 | 先锁定 1-3 个重点市场 |
| 目标买家 | 目标是进口商、批发商、品牌商、终端采购还是渠道商？ | 买家类型不清，导致名单混杂 | 明确买家类型和采购特征 |
| 目标产品 | 主推产品、利润产品、样品产品分别是什么？ | 触达内容和产品不匹配 | 先选主推产品和应用场景 |

## 第二步：跑 AIReach 自动开发

| 模块 | 动作 | 输出 |
|-|-|-|
| 生态画像挖潜 | 基于目标市场、买家类型、产品场景识别潜客 | 潜客池 |
| 自动背调 | 补全公司画像、联系人、业务相关度和潜在需求 | 高潜客户识别 |
| 千邮千面触达 | 按客户画像生成个性化内容并多轮触达 | 触达记录、打开、回复 |
| 持续孵化 | 对未立即回复但有价值的潜客持续跟进 | 待激活客户池 |
| 人工接管 | 对高意向回复和关键客户交给 FDE / CSM | 有效线索 |
| CRM 沉淀 | 有效线索入库、去重、分配、跟进 | 客户资产 |

## 第三步：看指标判断卡点

| 指标 | 判断什么 | 常见卡点 |
|-|-|-|
| 全网匹配数据量 | 市场上是否有足够潜客 | 市场太窄或关键词不准 |
| 已挖掘潜客数 | AIReach 是否找到足够目标客户 | 画像条件太宽或太窄 |
| 触达数 | 是否有足够开发动作 | 触达规模不足 |
| 打开率 | 邮件标题、发件身份、内容相关度 | 触达内容不吸引 |
| 回复率 | 买家需求匹配和内容说服力 | 买家不准或话术弱 |
| 高潜客户识别准确率 | 背调是否能识别真正值得跟进的客户 | 评分规则需要优化 |
| 孵化转化率 | 待激活客户是否被持续推进为有效线索 | 缺持续跟进或人工接管 |

## FDE 标准动作

1. 先和客户确认目标市场、买家类型、主推产品和优先场景。
2. 用 AIReach 建立第一批潜客池，不追求一开始就全市场覆盖。
3. 检查自动背调是否能识别公司画像、关键人和需求相关度。
4. 配置千邮千面触达内容，确保不是同一封开发信群发。
5. 观察打开率、回复率、高潜识别准确率和孵化转化率。
6. 把有效线索沉淀进 CRM，并标注来源、质量和下一步跟进动作。

## Bot 回答样稿

**新人提问**：AIReach 自动开发怎么判断有没有跑通？

**建议回答**：

不要只看发了多少封邮件。AIReach 要看完整闭环：是否找到目标潜客，是否完成自动背调，是否通过千邮千面触达，是否持续孵化，最后是否有有效线索进入 CRM。

你可以先拆三层指标：潜客挖掘精准度、自动触达规模、孵化转化率。  
如果潜客不准，先修目标市场、买家类型和产品画像。  
如果触达不足，检查触达规模、内容个性化和多轮触达节奏。  
如果有回复但没有形成客户，检查线索是否去重、入库、分配和跟进。

最终结果要落到 CRM 客户资产中，并交给“更高转化率”链路继续推进。
