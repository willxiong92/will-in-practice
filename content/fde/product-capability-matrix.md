> 一句话：SHOP · AIReach · CRM Lite/Smart/Pro · OMS · ERP API 的产品知识与场景映射。

## 定位

产品矩阵不是让新人背功能清单，而是让新人判断：客户当前经营问题应该用哪类产品或能力承接。

基本原则：

- 先判断客户经营象限，再匹配产品。
- 先讲业务作用，再讲功能入口。
- 产品不是孤立卖点，必须连接客户的获客、转化、客单价或毛利问题。

## 一张表看懂

| 经营公式 | 产品 / 能力 | 业务作用 | 新人判断问题 |
|-|-|-|-|
| 更多有效线索 | SHOP 官网独立站 | 承接官网流量、品牌展示、询盘沉淀 | 客户有没有稳定承接买家和转询盘的阵地？ |
| 更多有效线索 | AIReach 自动开发智能体 | 主动找客户、自动背调、批量触达、持续孵化 | 客户是否需要主动开发目标市场潜客？ |
| 更高转化率 | CRM Lite | 客户建档、跟进记录、基础管理 | 客户是否先需要把线索和客户资产管起来？ |
| 更高转化率 | CRM Smart | 商机推进、任务协同、过程可视、团队管理 | 客户是否要把销售过程和团队动作管起来？ |
| 更高客单价 | CRM Pro / OMS | 重点客户经营、组合销售、订单协同 | 客户是否需要围绕大客户、订单和协同做深经营？ |
| 更高毛利率 | CRM Pro / ERP API | 订单结构、报价纪律、系统打通和经营数据 | 客户是否需要提升报价、履约和跨系统效率？ |

## 三大产品线

| 产品线 | 解决什么问题 | 在团队 FDE-KB 里的落点 |
|-|-|-|
| 模块S / SHOP | 独立站建站、SEO/SEM 引流、询盘转化 | 更多有效线索 |
| 模块R / AIReach | 自动化客户开发、背调、触达、孵化 | 更多有效线索 |
| CRM | 客户管理、商机转化、团队协作、经营分析 | 更高转化率 / 更高客单价 / 更高毛利率 |

## CRM 版本理解

| 版本 | 一句话 | 适合客户 | FDE 服务重点 |
|-|-|-|-|
| Lite | 先把客户管起来 | 初创、单业务线、基础 CRM 需求 | 建档、跟进、基础客户资产 |
| Smart | 把团队销售过程管起来 | 成长型外贸团队，需要团队管理和过程可视 | 商机、任务、质检、团队数据 |
| Pro | 把外贸全流程管起来 | 多业务线、复杂协作、订单管理需求强 | 订单协同、数据大屏、复杂组织管理、系统打通 |

## 判断路径

1. 客户说“没线索”：先看 SHOP / AIReach。
2. 客户说“有线索但不成交”：先看 CRM Lite / Smart 的客户与商机链路。
3. 客户说“合作客户复购做不起来”：看 CRM Pro / OMS 的重点客户、订单和协同。
4. 客户说“利润低、报价乱、履约慢”：看 CRM Pro / ERP API 的流程、数据和系统打通。
5. 客户说“AI 能不能帮我”：先回到 AWB 经营大脑，判断要调用哪类 Skill / Agent。

## Bot 回答样稿

**新人提问**：OKKI 产品矩阵里 SHOP、CRM、AIReach、AWB 分别承担什么角色？

**建议回答**：

你可以按经营公式理解：SHOP 和 AIReach 主要解决“更多有效线索”。SHOP 是被动承接阵地，负责官网流量、品牌展示和询盘沉淀；AIReach 是主动开发能力，负责找客户、背调、触达和孵化。

CRM 主要解决“更高转化率”以及后续更高客单价、更高毛利率。Lite 偏基础客户管理，Smart 增强团队过程管理，Pro 承接订单协同、复杂组织和经营数据。

AWB 不是单一产品，而是经营大脑和 Skill / Agent 调度层，用来判断在选品、品牌、发品、营销、商机、成交、分析等环节该调用哪类 AI 能力。
