# B2B 搜索广告 MQL 清单：合格线索优先于点击量

B2B 广告的目标是 **MQL（营销合格线索）数量与成本**，不是点击或「询盘条数」本身。

## 1. 先写 MQL 标准

| 维度 | 合格示例 | 不合格示例 |
|---|---|---|
| 角色 | 采购/老板/工程决策相关 | 无角色、学生作业口吻 |
| 需求 | 具体产品/参数/场景 | 「你们做什么的」 |
| 主体 | 可识别的企业采购 | 纯个人消费意图 |
| 市场 | 目标国家/地区 | 明确非服务市场 |

- [ ] 团队书面定义 MQL  
- [ ] 落地页表单收集能判别的字段（职位/公司/需求描述等）  
- [ ] 销售反馈闭环：每周抽检「广告来的线索」合格率  

## 2. 流量质量闸门

- [ ] 审视是否关闭或收紧低质合作网络（按你使用的广告平台设置项）  
- [ ] 建立 **B 端否定词库** 并每月用搜索词报告补充  
- [ ] 常见排除方向：free/cheap/DIY/job/salary/thesis/pdf/used/rental 等（按行业裁剪）  
- [ ] 广告组级否定与受众排除，降低词义冲突  

## 3. 结构与落地

- [ ] 一组一主题，广告语与落地页同一承诺  
- [ ] 落地页有门槛：说清 MOQ/场景，过滤纯看热闹  
- [ ] 主 CTA 单一，表单字段够判别又不过度劝退  
- [ ] 与 [落地页结构](/practice/b2b-landing-page) 对齐  

## 4. 出价与匹配（原则）

- [ ] 先保证转化追踪正确，再谈智能出价  
- [ ] 广泛匹配必须配强否定与每周搜词清洗  
- [ ] 预算集中主推品，避免平均洒向弱页  

## 5. 周复盘

| 指标 | 看什么 |
|---|---|
| 花费 | 是否打在主推品 |
| 点击→有效对话 | 落地与匹配 |
| MQL 数与成本 | 是否接近团队目标 |
| 销售驳回原因 | 更新否定词与表单 |

## 完成标准

- MQL 定义销售认可  
- 连续两周能回答「钱主要花在哪类词/哪张页」  
- 不合格线索占比有下降动作记录  

## 风险边界

- 各广告平台菜单名不同；以当期后台为准  
- 不承诺 ROI；本页是经营检查表  
- 合规：广告与落地页声明真实，禁限售品不投
