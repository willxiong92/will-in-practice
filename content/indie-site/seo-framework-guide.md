# 独立站 SEO 基础框架：规划→执行→分析→优化

SEO 是长期自然流量与信任建设，不是「先狂发文章」。

## 何时该推进 / 何时不该急

**更适合开始：**

- 基础结构与关键页齐备（首页、产品、关于、联系）  
- 表单/询盘入口可用  
- 有持续内容与优化投入意愿  
- 目标市场有可识别搜索需求  

**不该急着谈起量：**

- 结构未定、产品说不清  
- 承接不稳（表单/邮箱失效）  
- 只要立刻询盘、零周期预期  
- 核心品类与市场未定  

此时先补结构与转化，再谈 SEO。参见 [增长闭环](/practice/growth-loop)、[询盘转化诊断](/practice/inquiry-conversion-diagnosis)。

## 闭环

```text
规划（定位与关键词主题）
  → 执行（技术基础 + 内容）
  → 分析（搜索与行为数据）
  → 优化（标题、内容、体验、内链）
  → 循环
```

## 规划

1. **给谁看、搜什么、与竞品差异**  
2. **关键词主题：** 短尾（首页/分类）+ 长尾（产品/文章）+ 语义相关词自然覆盖  
3. **页面分工：** 谁承载交易意图、谁承载认知意图  

从「堆词」转向「主题覆盖」：围绕核心主题做深，而不是同义堆砌。

## 执行顺序（避免先发文）

1. 站点结构与 URL 分工  
2. 关键词与意图映射到页面  
3. 核心产品页 / 分类 / FAQ  
4. 技术基础（见 [技术 SEO 清单](/practice/tech-seo-checklist)）  
5. 博客与扩展内容  
6. 内链、结构化数据、持续复盘  

先发文常见后果：无处承接、产品页弱、布局散、难复盘。

## 分析与优化

| 信号 | 优先动作 |
|---|---|
| 有展示点击低 | 优化 Title / Description |
| 有点击无询盘 | 内容意图与落地转化 |
| 收录/抓取异常 | 技术 SEO |
| 排名波动 | 内容质量与主题完整性 |

数据源原则：搜索表现 + 站点行为 + 竞品主题（工具名称会变，看你实际接入的后台）。

## 完成标准

- 能画一张「主题 → 页面 → 负责人」表  
- 能说明本站为何现在适合/不适合加大 SEO  
- 优化动作可回溯到数据信号，而非感觉  

## 风险边界

- 算法与界面年年变；框架稳定，技巧易朽  
- 不承诺排名与询盘数量
