# B2B 独立站 EEAT 内容检查清单

EEAT：Experience / Expertise / Authoritativeness / Trustworthiness。  
一句话：访客是否觉得「踏实、专业、不像骗子公司」。

## 四维落地

| 维 | 页面上怎么体现 |
|---|---|
| 经验 | 真实产线/工艺过程、可公开的合作过程说明、工程师视角 |
| 专业 | 参数表、工艺说明、认证与标准引用（有证据） |
| 权威 | 可核实的奖项、协会、专利、媒体（不编造） |
| 可信 | HTTPS、真实联系方式、关于我们、政策与隐私说明 |

## 四大死亡陷阱（先扫）

- [ ] **薄内容**：产品页只有图+价，无文字  
- [ ] **搬运**：从平台或其他站复制  
- [ ] **工厂英语**：错误多、读不懂  
- [ ] **自嗨**：「best/No.1」无证据  

## 首页

- [ ] 5 秒内看懂你是谁、卖什么价值  
- [ ] 主推品/能力露出  
- [ ] 信任元素（认证/客户类型/数据，须真实）  

## 产品页（权重最高）

- [ ] 标题：主词 + 关键特性/场景  
- [ ] 3–5 个**价值向**卖点（非纯功能罗列）  
- [ ] 问题—方案叙述 + 参数表  
- [ ] 应用场景具体  
- [ ] 可关联的案例/资料/视频（授权范围内）  
- [ ] FAQ 回答真实买家问题  
- [ ] CTA 清晰，无虚假倒计时  

## 语言与结构

- [ ] You-attitude 优先于 We-attitude  
- [ ] 结论先行，小标题与列表可读  
- [ ] 图有 alt；信息图优先于大段墙  
- [ ] 多语言：人工审校；各语言独立 TDK；文化信任元素本地化  

## 与 AI 生产

- [ ] 定调表完整（见 [AI 内容起草](/practice/ai-content-draft-sop)）  
- [ ] 无编造数据/案例  
- [ ] 终稿有人注入的经验句  

## 完成标准

- 主推产品页通过本清单抽检  
- 陷阱四项为 0  
- 更新记录含事实来源（内部）  

## 风险边界

- 质量评估算法会变；诚实与可验证是长期策略  
- 客户 logo/证言必须授权
