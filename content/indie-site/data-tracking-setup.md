# 独立站数据追踪：打通 GSC、GA4、广告转化与 MQL 对账

## 场景与问题

**读者**：独立站运营、外贸营销、负责站点交付验收的 FDE / CSM。

**场景**：新站上线、准备投放、月度复盘，或客户质疑「网站/广告没效果」。

**痛点**：

- 访问量在涨，却说不清询盘从哪来  
- 广告后台有转化，销售说线索差或根本没收到  
- SEO、广告、改版各自报数，口径对不上  
- 只装了统计代码，仍无法做业务复盘  

## 核心判断

数据追踪的目标不是「装几个代码」，而是让团队稳定回答四件事：

```text
客户从哪里来？ → 看了什么？ → 有没有提交询盘？ → 是不是有效 MQL？
```

最低可用闭环是：**GSC + GA4 +（如有）广告主转化 + UTM + 表单来源字段 + CRM/MQL 对账**。  
没有 MQL 对账时，只能说明流量和表单层，不能说明业务质量。

## 方法拆解

### 1. 先认清四层数据各自回答什么

| 层级 | 常见工具 | 回答的问题 |
|---|---|---|
| 搜索入口 | Google Search Console（GSC） | 有没有被搜到、哪些查询/页面 |
| 站内行为 | Google Analytics 4（GA4） | 从哪来、看了什么、关键动作有没有发生 |
| 广告成本 | 搜索广告后台 | 钱花在哪、转化是否真实 |
| 业务结果 | 表单 / CRM / 线索表 | 是谁、是否 MQL、是否被跟进 |

### 2. 新手最小闭环：先打通 6 点

| 点 | 对象 | 完成标志 |
|---|---|---|
| 1 | GSC | 已验证；sitemap 可读 |
| 2 | GA4 | Realtime 可见真实访问 |
| 3 | 广告主转化（若投放） | 主转化 = 有效业务动作，非虚荣指标 |
| 4 | 表单来源字段 | 能记 UTM / 着陆页 / referrer |
| 5 | 线索入库 | CRM 或线索表能看到进入 |
| 6 | MQL 标注 | 团队书面定义且可抽检 |

### 3. 执行分层

**P0 上线前必须完成**

1. **GSC**：资源验证、提交 sitemap、`robots` 未误屏蔽核心页、核心页具备可索引技术条件  
2. **GA4**：属性对应正确站点、Realtime 可用、关键事件至少覆盖表单成功提交（可选即时通讯/邮箱/电话点击）、内部流量可识别  
3. **广告主转化（如有）**：表单成功作主转化；点击类按业务需要作次级或并列；页面浏览/曝光/滚动不作主转化  
4. **表单隐藏字段**：至少 `utm_source/medium/campaign`、`landing_page`、`referrer`；广告点击 ID 按平台能力保留；字段须进入线索详情或备注  

**P1 投放与复盘前补齐**

- UTM：全小写、不用空格、活动名可读、复盘前不随意改命名  
- 对账指标：表单数、可联系线索、MQL 数、MQL 率、首回 SLA；有 SQL/成交再纳入  

**P2 进阶（有能力再做）**

- 增强型转化、价值回传、看板：需稳定字段与配置能力  
- 隐私同意相关能力：以目标市场法规与广告平台当期官方文档为准（本篇不写死细则）  

### 4. 现象分流（诊断）

| 现象 | 先查 |
|---|---|
| 广告有点击，GA4 无访问 | 标签、跳转丢参、装错站、Realtime |
| GA4 有转化，CRM 无线索 | 真实提交、入库与事件是否误触发 |
| 来源全是 direct | UTM 与隐藏字段 |
| 转化暴涨 | 重复触发、弱事件当主转化、垃圾询盘 |
| CPA 好看但销售不满 | MQL 率与驳回原因 |

## 风险边界

**适用**：有可访问站点、可提交表单、能约定 MQL、能协调标签/域名权限。

**不适用**：

- 只想「装一个代码就算完」  
- 无线索接收人，也无 CRM/线索表  
- 需要具体隐私法规条文解释（应另查目标市场合规）  
- 指望本页承诺 CPA、ROI 或排名周期  

**常见错误**：

- 把页面浏览设成主转化  
- 字段记在表单里但进不了线索库  
- 没有 MQL 定义却用转化数谈效果  
- 把未核验的隐私/Consent 细则当固定流程抄进团队 SOP  

**易变事实**：GSC / GA4 / 广告后台菜单与功能名会变；以各产品当期官方界面为准。

## 行动工具

### 工具 A：新站交付验收清单

- [ ] GSC 已验证并提交 sitemap  
- [ ] robots 未屏蔽核心页  
- [ ] GA4 Realtime 可见访问  
- [ ] 表单成功可触发关键事件  
- [ ] 若投放：广告主转化已测通  
- [ ] 表单隐藏字段可记录来源与着陆页  
- [ ] 线索能进入 CRM 或线索表  
- [ ] 内部测试流量可识别  

### 工具 B：广告投放前验收

- [ ] 主转化 = 有效询盘类动作  
- [ ] 带 UTM 的测试链参数不丢  
- [ ] 成功提交不重复计数  
- [ ] CRM 能区分广告 / 自然 / 其他来源  

### 工具 C：月度复盘一页纸（模板）

```markdown
# 独立站数据追踪复盘 · YYYY-MM

- 最大问题：追踪缺失 / 流量少 / 转化低 / MQL 差 / 跟进断层

## 数据闭环
| 层级 | 关键指标 | 本月 | 上月 | 判断 |
|---|---|---:|---:|---|
| GSC | 自然搜索点击 / 核心收录 |  |  |  |
| GA4 | 访问 / 关键事件 |  |  |  |
| Ads | 花费 / 转化 / 单位成本 |  |  |  |
| CRM | MQL / MQL 率 / 首回 |  |  |  |

## 问题定位
| 问题 | 证据 | 影响 | 优先级 |
|---|---|---|---|

## 下月动作（不超过 3 个）
- [ ] P0：

- [ ] P1：
```

### 验收标准

- 能稳定回答四个问题：从哪来、看了什么、有没有询盘、是不是 MQL  
- 新站 / 投放前清单可勾选通过  
- 连续两个月能用同一口径做来源 × 询盘 × MQL 对照  
- 下月动作可收敛到 3 个以内且有负责人  

**缺口标注**：Consent Mode / 增强型转化 / 价值回传的官方细则未在本文展开（`uncertain`，待官方文档核验后再单篇处理）。
