# 独立站 SEO 基础诊断（30 分钟）

SEO 诊断不要从“写多少文章”开始。先判断网站能否被抓取、被理解，以及能否支撑询盘。

## 诊断前准备

- 官网域名
- 目标国家/地区
- 核心产品
- 能拿到的 GSC / GA4 权限
- 询盘路径（表单 / 邮箱 / WhatsApp）

没有权限时：用 `site:` 粗看收录、PageSpeed 看速度、手动检查核心页面与结构化数据。

## Step 1：致命红线

| 检查项 | 红线信号 | 优先级 |
|---|---|---|
| HTTPS | 证书异常、混合内容 | P0 |
| 域名规范化 | www / 非 www 多版本并存 | P0 |
| robots.txt | 误屏蔽核心目录 | P0 |
| sitemap | 无 sitemap 或大量无效 URL | P0 |
| 索引 | 核心产品页未收录 | P0 |
| 转化入口 | 用户无法提交询盘 | P0 |

有 P0 就先修基础，不先讨论外链和批量内容。

## Step 2：技术基线

- 速度与 Core Web Vitals 是否明显异常
- 移动端是否可读可点
- 是否有基础结构化数据
- 是否存在大量 404
- 重要产品页是否在合理点击深度内

## Step 3：内容与意图

| 页面类型 | 应回答的问题 |
|---|---|
| 首页 | 我是谁、做什么、服务谁、为什么可信 |
| 产品页 | 规格、场景、差异、下一步如何询盘 |
| 分类/解决方案页 | 覆盖哪类采购意图 |
| 博客 | 是否服务主题权威，而不是无效堆文 |

## Step 4：输出优先级

只输出 3 条优先动作：

1. 必须立刻修的红线
2. 最影响主推产品被理解的内容问题
3. 最影响转化的页面或入口问题

## 完成标准

- 30 分钟内形成可沟通的诊断结论
- 客户知道先做什么，而不是收到 20 条建议清单
- 技术红线与内容问题被区分开

## 风险边界

SEO 结果受竞争、站龄、行业和资源投入影响。本 playbook 用于诊断与优先级排序，不承诺排名周期。
