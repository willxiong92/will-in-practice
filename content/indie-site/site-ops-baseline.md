# 独立站建站与运维基线：上线前与月度巡检清单

## 场景与问题

**读者**：独立站运营、建站协同、做交付验收的 FDE / CSM、需要统一上线口径的负责人。

**场景**：新站上线前验收、改版后复验、接手客户站点首次体检、建立月度巡检。

**痛点**：

- 页面「看起来正常」，实际丢询盘或邮箱收不到  
- 上线后无人巡检，出故障才救火  
- 技术、SEO、数据、安全责任混在一起，没有合格线  
- 过早讨论增长，却连可达与可复盘都没做到  

## 核心判断

独立站交付不是「页面能打开」就结束。  
最低基线只有一句话：**可访问 · 可询盘 · 可收录 · 可复盘**。  
未过线不宣称交付完成；测速分数不等于经营效果。

## 方法拆解

### 1. 六层基线

| 层级 | 核心问题 | 失败后果 |
|---|---|---|
| 域名 / DNS | 能否稳定访问 | 全站不可用 |
| HTTPS / CDN | 是否安全、可访问 | 信任下降、速度差 |
| 邮箱 / 表单 | 询盘能否收到 | 直接丢线索 |
| SEO 技术 | 能否被抓取索引 | 有站无收录 |
| 数据追踪 | 能否复盘来源与转化 | 不知道效果 |
| 安全 / 备份 | 出问题能否恢复 | 篡改、数据丢失 |

### 2. 合格线分层

| 层级 | 标准 |
|---|---|
| 基础合格 | 能打开、表单可达、GSC/GA4/基础转化可装、主页面移动端可用 |
| 较优 | 速度可接受、TDK/sitemap/robots/基础 Schema 完整、UTM 与来源字段可记 |
| 进阶 | 备份恢复演练清楚、月度巡检稳定、结构化能力按需补齐 |

### 3. 工具分工（避免错用）

| 工具类型 | 用途 | 不该被当成 |
|---|---|---|
| 页面测速 / 体验检查 | 上线前技术基线 | 经营效果证明 |
| GSC | 收录与搜索入口 | 唯一 SEO 结论 |
| GA4 | 访问与事件 | 替代 CRM 业务质量 |

### 4. 巡检节奏

| 频率 | 检查项 |
|---|---|
| 每周 | 主页面访问、表单真实提交、询盘通知是否到达 |
| 每月 | GSC 索引异常、404、速度抽检、转化追踪、垃圾询盘 |
| 每季度 | Schema 抽检、核心页转化、备份恢复演练、权限清理 |

### 5. 故障分流

| 故障 | 先做什么 |
|---|---|
| 网站打不开 | 先 DNS，再主机/CDN/证书 |
| 表单收不到 | 真实提交 + 查邮件/后台/入库 |
| GA4 无数据 | Realtime 与标签调试 |
| GSC 难收录 | 先技术（robots/noindex/sitemap）后内容 |
| 页面变慢 | 大图、缓存、第三方脚本 |
| 疑似被篡改 | 止损、恢复备份、改权限 |

## 风险边界

**适用**：有明确权限归属（域名、后台、邮箱、表单接收人），团队接受「基线未过线不算交付完成」。

**不适用**：深度定制开发与企业 IT 架构设计；已发生严重安全事件时的专业应急响应（先止损）；代替 SEO 排名或广告效果报告。

**常见错误**：

- 只测首页打开，不测真实表单端到端  
- 用测速分数代替询盘可达  
- 域名/邮箱权限只在客户侧却承诺「全部运维包干」  
- 把内部服务版本话术直接对客承诺  

**说明**：SPF/DKIM/DMARC 等发信认证以邮箱服务商当期说明为准，本清单只要求「按需配置、可收发」，不写死厂商菜单。

## 行动工具

### 工具：上线前验收清单

**域名、DNS、HTTPS**

- [ ] 域名账号与解析权限归属清楚  
- [ ] 主域名、`www`、必要子域名解析正确  
- [ ] 全站 HTTPS，无证书错误  
- [ ] `www` / 非 `www` 统一 301，无跳转环  
- [ ] 目标市场访问基本可用（含 CDN 时抽检）  

**企业邮箱与询盘入口**

- [ ] 企业邮箱可正常收发  
- [ ] 发信认证按需配置（如 SPF/DKIM/DMARC）  
- [ ] PC / 移动端表单均可成功提交  
- [ ] 表单通知到达约定邮箱或 CRM  
- [ ] 即时通讯 / 邮箱 / 电话等入口可点  
- [ ] 有基础垃圾询盘防护  

**SEO 技术基础**

- [ ] `robots.txt` 未屏蔽核心页与必要资源  
- [ ] `sitemap.xml` 含核心页，可提交 GSC  
- [ ] 首页、分类、产品页有独立基础 TDK  
- [ ] 按需基础结构化数据（如 Organization / Product / FAQ）  
- [ ] 重要旧 URL 有跳转或自定义 404  
- [ ] 移动端可读、主 CTA 可点  

**数据追踪（最小）**

- [ ] GSC 已验证，sitemap 已提交  
- [ ] GA4 Realtime 能看到访问  
- [ ] 若投放：广告主转化可触发  
- [ ] UTM 与表单来源字段可记录  

**安全与备份**

- [ ] 后台强密码 / 双因素（平台支持时）  
- [ ] 系统与扩展可更新，无已知高危未处理项（按平台能力）  
- [ ] 备份位置、频率、恢复责任人写清楚  
- [ ] 账号清单与交付说明已交接  

### 验收标准

- 上线前清单全部勾选，或未勾选项有书面风险与负责人  
- 至少一次真实表单端到端测试（提交 → 通知 → 入库/记录）  
- 月度巡检有日期、检查人、异常与处理记录  
- 团队能一致回答站点是否「可访问、可询盘、可收录、可复盘」
